Path Sum III

发表于 2017-04-14 | 分类于 算法设计 |

The Question

You are given a binary tree in which each node contains an integer value.

Find the number of paths that sum to a given value.
The path does not need to start or end at the root or a leaf, but it must go downwards (traveling only from parent nodes to child nodes).
The tree has no more than 1,000 nodes and the values are in the range -1,000,000 to 1,000,000.

The Example

root = [10,5,-3,3,2,null,11,3,-2,null,1], sum = 8

      10
     /  \
    5    -3
   / \     \
  3   2     11
 / \   \
3  -2   1

Return 3. The paths that sum to 8 are:

  1. 5 -> 3
  2. 5 -> 2 -> 1
  3. -3 -> 11

给定一颗二叉树，给定一个特定的值sum，寻找树中的路径条数，使得路径上的数字的和为sum

The Answer1

本题求解首先要明白题目的意思，路中任意一个满足条件的路径都可以，而不仅限于出发点必须是树的根节点。
刚开始是如下所示的解题思路：

  1. 得到树中从根节点到叶节点的所有路径，使用DFS；
  2. 在路径中搜索满足条件的子路径；
  3. 在路径中遍历求解时，记录从每一个节点开始出发的路径和，只要满足sum的要求就是一个路径；
  4. 记录每一个出发点需要使用双层循环加数组的形式。
    但是后来发现这个解题方法中存在一个问题：根节点到叶节点的路径有重复现象，如果满足条件的子路径就是出现在重复路径上，那么就会重复计算，这就需要去除重复的路径。

The Answer2

基于第一个想法继续思考怎么去重复呢，后来发现代码量有点大，就反思方法必然出现了问题，如何解决问题呢？
自然就想到了子路径必须是从根节点出发的路径，那又如何保证路径可能是从任以节点出发的情况呢？那就不止求从root出发的可能路径，还要求树的所有子树中的路径数目。来解决子路径必须包含根节点的情况。

  1. 求从root出发的路径满足条件的子路径数目；
  2. 递归求解所有子树，从子树根节点出发的所有子路径的数目；
  3. 求解过程中任然使用DFS求解。
