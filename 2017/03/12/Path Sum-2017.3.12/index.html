存在的问题

使用if遍历，虽然边界条件已经设置好了，但是总会出现如下错误：

思考了好久算法这样写的话，就会存在一个问题：树中的最左边的第一条路径如果遍历完成，函数就会返回一个值，不管是正确还是错误，结束调用，就不会走右边的路径了。这就是问题所在了。

而改进就可以可以使用或选择，即使左边的路径遍历玩，只要结果不是sum就仍然会遍历右边的路径，直到找到一条路径为sum或者将树便利完。
